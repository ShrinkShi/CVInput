# 开发者调试

[返回 README](../README.md)

CVInput 内置开发者模式，用于排查窗口定位、DPI、换行行为、输入法切换等问题。

## 开启方式

1. 打开设置窗口。
2. 开启“开发者模式”。
3. 设置窗口旁会出现“开发者调试”窗口。
4. 按需开启日志分类。

## 日志分类

| 分类 | 配置项 | 说明 |
|---|---|---|
| 窗口位置调试日志 | `debug_window_position` | 记录设置、关于等子窗口尺寸、坐标、DPI、跟随移动数据 |
| 换行行为调试日志 | `debug_newline_behavior` | 记录输入模式、换行处理、底层 `Shift+Enter`、IME 切换数据 |

## 导出日志

开发者调试窗口提供：

- 导出调试日志；
- 清空调试日志；
- 当前日志条数显示。

日志导出编码由设置项“输出编码”控制，默认 `UTF-8`。

## 常用排查点

### 子窗口位置异常

开启“窗口位置调试日志”，重点查看：

- 主窗口 geometry；
- 子窗口 target geometry；
- `winfo_x/y`；
- `winfo_rootx/rooty`；
- 屏幕宽高；
- DPI awareness 是否已在启动最早阶段执行。

### 换行异常

开启“换行行为调试日志”，重点查看：

- 当前输入模式；
- 最终候选文本是否包含尾部换行；
- 默认模式下底层换行实现；
- 单行模式是否已替换换行；
- 分裂模式队列是否正确推进。

### 输入法异常

同样使用“换行行为调试日志”，重点查看：

- 输入前目标窗口句柄；
- previous layout；
- target layout；
- 切换请求是否成功；
- 输入后是否恢复原布局。
