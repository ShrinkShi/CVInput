# CVI

CVI 是一个 Windows 桌面小工具，用于把剪贴板文本转换为逐字符模拟输入。复制文本后，程序会自动读取剪贴板内容，并在主界面同时显示“原始剪贴板文本”和“待输入候选文本”；点击目标输入框后，按全局快捷键或点击输入图标即可把候选文本逐字符输入到当前焦点窗口。

## 特点

- 使用 `customtkinter` 构建小尺寸、深色、无边框自定义标题栏界面。
- 默认全局快捷键为 `Ctrl+V`，支持自定义快捷键。
- 支持主界面热键开关，默认 `Ctrl+Alt+V` 可快速启用/禁用输入热键。
- 使用 Windows 原生 `RegisterHotKey` + `GetMessageW` 监听全局快捷键。
- 使用 `pyperclip` 后台轮询剪贴板。
- 使用 `pynput.keyboard.Controller` 逐字符模拟输入，支持中文、英文和常见符号。
- 支持系统托盘、关闭时最小化到托盘、托盘单击显示窗口和托盘退出。
- 支持开机自启、窗口置顶、透明度、输入后清空、中英文语言切换、记住设置和恢复默认设置。
- 支持空剪贴板快捷键释放保护、输入前切换英文输入法并在输入后恢复、三种输入模式、三种输入间隔模式、开发者调试日志、集成使用说明的关于页面和 10 个 Alt+数字多槽位输入。

## 安装依赖

建议使用 Python 3.10 或 3.11，在 Windows 上运行：

```bash
python -m venv .venv
.venv\Scripts\activate
python -m pip install -r requirements.txt
```

## 运行方式

```bash
python main.py
```

运行后窗口默认置顶，默认会自动监听剪贴板，并在系统托盘创建图标。多槽位输入和关闭时最小化到托盘默认开启。

## 打包方式

```bash
python -m PyInstaller --onefile --windowed --name CVInput --icon assets/icon.ico --add-data "src/locales;src/locales" --add-data "assets;assets" main.py
```

打包产物会生成到 `dist/`，构建缓存会生成到 `build/`。`--add-data "src/locales;src/locales"` 用于包含语言文件，`--add-data "assets;assets"` 用于包含标题栏的 `set.png`、`about.png` 等运行时图标。这些构建目录不应提交到仓库。

## 快捷键

默认快捷键是 `Ctrl+V`。当 CVI 成功注册该快捷键时，按下 `Ctrl+V` 会触发逐字符模拟输入。输入前程序会释放触发快捷键的按键，避免 `Ctrl`、`Alt` 等修饰键残留影响后续输入。

点击标题栏左侧的设置图标，设置弹窗会贴在主窗口左侧打开。在“输入快捷键”输入框里填写新的快捷键，然后点击输入框右侧的应用图标。

主界面右侧有“热键”开关。关闭后，输入快捷键和多槽位 `Alt+数字` 快捷键都会停用，但输入、停止、读取等功能按钮仍可使用；剪贴板监听只受“监听”开关控制。

“热键开关快捷键”默认是 `Ctrl+Alt+V`，始终保持注册，用于快速启用或禁用热键。即使主热键开关已经关闭，也可以按 `Ctrl+Alt+V` 重新打开。

支持示例：

```text
Ctrl+V
Ctrl+Alt+V
Ctrl+Alt+S
Alt+V
Shift+F8
Ctrl+Shift+Enter
```

支持的修饰键包括 `Ctrl / Control`、`Alt`、`Shift`、`Win / Windows / Meta`。支持的主键包括 `A-Z`、`0-9`、`F1-F24`、`Enter`、`Tab`、`Space`、`Esc`、`Home`、`End`、`PageUp`、`PageDown`、`Delete`、`Backspace` 和方向键。

## 空剪贴板保护

设置项“剪贴板无文本时不接管快捷键”默认开启。开启后，如果主文本框为空或只有空白字符，程序会注销主快捷键，让 `Ctrl+V` 或自定义主快捷键回归目标软件的原生处理；主文本再次出现有效内容时会自动重新注册快捷键。

该保护只影响主快捷键，不影响主界面的输入图标，也不影响多槽位 Alt+数字输入。

## 自动切换英文输入法

设置项“输入前后切换英文输入状态”默认开启。程序会在输入开始前记录当前前台目标窗口的键盘布局，如果不是英语美国布局 `0x0409`，则通过 `WM_INPUTLANGCHANGEREQUEST` 请求目标窗口切换到英文布局；输入结束后会尽量把同一个目标窗口恢复到输入前记录的布局。

该能力依赖 Windows 已安装英文输入法/键盘布局，并依赖目标窗口接受输入语言切换请求。检测或切换失败时程序不会崩溃；开启开发者模式后可在换行行为调试日志中看到输入法检测、切换和恢复结果。

## 双文本框架构

主界面上方文本框是“原始剪贴板文本”，用于保留剪贴板读到的原始内容。下方文本框是“待输入候选文本”，实际模拟输入只读取候选文本。

不同输入模式会根据原始文本生成候选文本。用户也可以手动编辑候选文本；本次快捷键输入会以候选文本为准。

## 输入模式

设置中的“模式”提供三种用户侧策略：

- 默认模式：遇到换行时模拟 `Shift+Enter`，适合普通输入框和能够正确处理组合键换行的文本框。
- 单行模式：将换行替换为可选的空格或 `Tab`，适合 AI 输入框、搜索框和不要求严格保留分行格式的场景。
- 分裂模式：读取剪贴板时按行创建待输入队列，主文本框每次只显示当前一行；每完成一次输入，自动推进到下一行。

微信、QQ 等聊天软件可能不会把程序模拟的 `Shift+Enter` 解释为换行，甚至仍会触发发送。对于这些高风险输入框，建议使用分裂模式逐行确认输入，或使用单行模式消除换行。

开发者模式中仍保留底层换行实现选择，包括 `pynput`、`win32_scan`、`win32_vk`、`pyautogui` 和 `keyboard`，用于复测和日志分析；这些选项不作为普通用户模式展示。普通 ASCII 字符使用现有输入引擎逐字符输入，中文等非 ASCII 字符优先走 Windows Unicode 输入路径。

## 输入间隔模式

设置中的“输入间隔模式”提供三种策略：

- 默认模式：使用项目默认输入间隔，当前为 `30ms`。
- 自定义间隔模式：显示“输入间隔(ms)”输入框，按固定字符间隔输入，范围沿用 `0.1ms ~ 10000ms`。
- 完成时间模式：显示“完成时间(ms)”输入框，范围为 `1ms ~ 10000ms`。程序会按 `完成时间 / 本次实际候选文本字符数` 动态计算每字符间隔。

例如 100 个字符、完成时间 `1000ms`，每字符约 `10ms`；10 个字符、完成时间 `1000ms`，每字符约 `100ms`。极短完成时间会受 Windows 调度精度、目标输入框处理能力、输入法和权限状态限制，不保证精确达到。

## 多槽位输入

开启设置项“多条复制内容手动输入”后，主窗口会显示包含 10 个紧凑多行文本框的滚动区域：

```text
Alt+1
Alt+2
Alt+3
Alt+4
Alt+5
Alt+6
Alt+7
Alt+8
Alt+9
Alt+0
```

按对应热键会用同一套 `TypingEngine` 输入对应槽位内容。槽位内容会持久化保存。多槽位热键注册失败不会影响主快捷键。

## 托盘与关闭行为

程序运行时会创建系统托盘图标。单击托盘图标会显示并置前主窗口；右键菜单包含：

- 显示/隐藏窗口
- 退出

“关闭时最小化到托盘”默认开启。开启时，点击主窗口关闭按钮只会隐藏窗口，快捷键和剪贴板监听继续运行。此时需要通过托盘菜单“退出”才会真正关闭程序。

如果关闭该选项，点击关闭按钮会直接退出程序，并注销全局快捷键、停止剪贴板后台线程和托盘图标。

## 开机自启

设置中的“开机自启”使用当前用户的注册表 Run 项：

```text
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run
```

开启时写入 `CVInput` 项，关闭时删除该项。操作失败时程序会在状态栏提示，不会崩溃。

## 语言切换

设置中可以在中文和英文之间切换。语言文件位于：

```text
src/locales/zh_cn.json
src/locales/en_us.json
```

缺失语言 key 时会回退到中文默认值或 key 本身，不会导致程序崩溃。

## 设置保存

设置项“下次启动时记住设置项”默认开启。开启时，用户修改的配置会写入本地配置文件，下次启动自动恢复；关闭时，本次运行内的修改仍会临时生效，但配置文件只保留“不记住设置”标记，下次启动会恢复默认配置。

设置项“子窗口点击空白处关闭”默认关闭。开启时，设置窗口和关于窗口失去焦点、点击桌面空白处或切换到其他软件时会自动关闭；关闭时，子窗口只通过右上角关闭按钮关闭。设置窗口和关于窗口会贴在主窗口左侧，并在主窗口拖动时跟随移动。

## 开发者模式

设置中的“开发者模式”默认关闭，不影响普通用户使用。开启后，设置窗口左侧会出现“开发者调试”子窗口，用于按分类控制调试日志。

当前支持：

- 窗口位置调试日志：记录设置、关于等子窗口的尺寸、坐标、DPI 和跟随移动数据。
- 换行行为调试日志：记录输入过程中换行采用组合键、单行替换或分裂推进策略，以及底层按键实现和按下/释放顺序。

开发者调试窗口提供“导出调试日志”和“清空调试日志”。导出文件使用设置中的输出编码，默认 UTF-8，包含当前开发者模式状态、分类开关状态和最近的调试记录。

## 恢复默认设置

设置弹窗中提供“恢复默认设置”按钮，会恢复输入快捷键、热键开关快捷键、热键开关、输入间隔模式、默认 `30ms` 输入间隔、默认 `1000ms` 完成时间、自动监听剪贴板、输入后清空、关闭到托盘、开机自启、透明度、语言、置顶状态、空剪贴板保护、输入法切换保护、默认输入模式、单行替换字符、底层换行实验实现、记住设置、子窗口关闭行为、多槽位设置和开发者调试设置。

## 快捷键注册失败怎么办

Windows 全局快捷键同一时间只能被一个程序注册。如果 `Ctrl+V` 或你设置的其他快捷键已经被系统、输入法、远程桌面软件、安全软件或其他程序占用，CVI 就无法注册该快捷键。

注册失败时程序不会崩溃，状态栏会提示“输入快捷键注册失败”或“热键开关快捷键注册失败”。此时仍可点击输入图标手动触发输入，也可以打开设置，把快捷键改成 `Ctrl+Alt+V`、`Ctrl+Alt+S`、`Alt+V` 等组合。

## 为什么某些目标窗口需要管理员权限

Windows 有权限隔离机制：普通权限运行的程序通常不能向管理员权限运行的窗口发送模拟键盘输入。例如，目标窗口是以管理员身份启动的命令行、安装器或系统管理工具时，普通权限的 CVI 可能无法输入。

如果需要向管理员权限的目标窗口模拟输入，请右键以管理员身份运行 CVI，使 CVI 和目标窗口处于相同或更高权限级别。

职责说明：

- `main.py`：程序入口，只创建并启动 `CVInputApp`。
- `src/app/app.py`：组合 UI、快捷键、剪贴板、输入引擎、托盘和配置，保持主协调类尽量薄。
- `src/app/*_controller.py`：按热键、剪贴板、输入、设置、托盘和生命周期拆分应用协调逻辑。
- `src/ui/main_ui.py`：customtkinter 无边框主界面与当前窗口 UI 组合入口。
- `src/ui/tooltip.py`：Tooltip 生命周期和安全销毁。
- `src/ui/theme.py`：UI 颜色、主题常量。
- `src/ui/assets.py`：PyInstaller 兼容资源路径。
- `src/ui/window_utils.py`：窗口存在性检查等通用工具。
- `src/ui/settings_ui.py`、`src/ui/about_ui.py`、`src/ui/slot_ui.py`：设置、关于、多槽位 UI 的后续精细拆分入口。
- `src/hotkey.py`：Windows `RegisterHotKey` 多热键监听。
- `src/clipboard.py`：`pyperclip` 剪贴板轮询。
- `src/typing_engine.py`：逐字符模拟输入、换行策略和非 ASCII 字符输入分发。
- `src/win_input.py`：以 Windows `SendInput` 的虚拟键或扫描码模式注入 `Shift+Enter` 等特定组合键事件。
- `src/ime.py`：Windows 目标窗口键盘布局检测、输入前切换英文布局与输入后恢复。
- `src/config.py`：JSON 配置读写。
- `src/i18n.py`：语言文件加载与 fallback。
- `src/tray.py`：系统托盘图标与菜单。
- `src/startup.py`：Windows Run 项开机自启。

## 使用建议

1. 复制要输入的文本。
2. 确认 CVI 文本框里已经显示该文本。
3. 点击目标程序的输入框，使目标输入框获得焦点。
4. 按 `Ctrl+V` 或你设置的快捷键。
5. 如需中断输入，点击停止图标。
6. 如需查看使用说明和注意事项，点击标题栏左侧的关于按钮。

## 注意事项

- 输入过程中不要切换焦点，否则文本会继续输入到新的焦点窗口。
- 如果要手动编辑 CVI 文本框，程序在文本框获得焦点时不会用剪贴板内容覆盖它。
- 极短输入间隔或极短完成时间会受系统调度精度、目标程序处理能力和输入法状态限制。
- 微信、QQ 等聊天软件对模拟 `Shift+Enter` 的处理可能不同；如果默认模式换行不稳定，建议使用单行模式或分裂模式。
- 输入法切换依赖系统已安装英文输入法/键盘布局；如果目标窗口以管理员权限运行，CVI 也可能需要以管理员身份运行。
- 托盘“退出”和关闭按钮的行为不同：开启“关闭时最小化到托盘”后，关闭按钮只隐藏窗口，托盘“退出”才会释放资源并结束程序。
